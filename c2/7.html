상수 const
상수는 변하지 않는 수를 뜻합니다.

프로그래밍에서 상수가 왜 필요 할까요?

프로그래밍은 코드를 작성할 수록 복잡해지고, 그 구조가 매우 복잡해집니다.
그렇다면 본인이 작성 해놓고 깜빡하는 변수들이 생기죠. 그러다 그 변수를 건드리면?
전체적인 오류가 나고 이를 잡기 위해 디버그를 하겠죠. 이러한 오류를 사전에 방지하려고 상수를 사용합니다.

상수는 실수를 방지하기 위해 사용


상수의 키워드는 const 입니다.

const 자료형 상수명 = 값

상수로 지정된 변수는 어떤짓을 해도 변경이 불가능합니다.
하지만 C# 에 강제로 바꿀 수 있게 해주는 기능이 있다면 변경이 되겠죠.

상수가 쓰이는 곳은 주로 값의 변경이 이루어 지지 않아야 하는 부분에서
많이 사용이 됩니다.

예를 들어 매개변수로 받는 값이라던지요..


읽기 전용 상수 readonly
우리가 알던 상수는 컴파일러가 걸러주는데 읽기 전용 상수는 런타임에서 상수의 지정이 가능합니다.
기존의 상수보다 아주 유연한 녀석이죠.



각 클래스 Con, Read는 const와 readonly 타입을 가지고 있습니다.
위의 그림과 같이 const로 지정된 멤버 변수를 가지는 Con은 컴파일러가 이미 걸러서 수정이 불가능합니다.
하지만 Read는 런타임에서 설정을 하고 그 이후에 가지는 값을 상수로 가집니다.

너무 딱딱한 const 보다는 readonly를 사용해보세요.

열거형
이 열거형은 상수를 순서대로 나열해서 선언을 해줍니다.
const 를 안붙히고 한번에 선언 한다고 생각하세요.

그리고 이 열거형에는 특징이 하나 있는데요.
선언시 자료형을 지정해주지 않으면 자동으로 int형을 부여해줍니다.

그리고 사용 가능한 자료형은 정수 형식의 자료형만 사용이 가능합니다.
byte, sbyte, short, ushort, int, uint, long, ulong, char

enum 열거 형식명 : 자료형 { 상수1, 상수2, 상수3 ... }


class HelloWorld
{
    enum Dialog { YES, NO }
    enum Dialog2 : sbyte { YES = 2, NO = 4 }
    static void Main(string[] arg)
    {
        Console.WriteLine("{0} {1}", (int)Dialog.YES, (int)Dialog.NO);
        Console.WriteLine("{0} {1}", (sbyte)Dialog2.YES, (sbyte)Dialog2.NO);
        return;
    }
}



using System;
using static System.Console;

namespace CSharp
{
    class HelloWorld
    {
        enum Dialog { YES, NO }
        static void Main(string[] arg)
        {
            Dialog result = Dialog.YES;
            Console.WriteLine("{0}", result == Dialog.NO);
            Console.WriteLine("{0}", result == Dialog.YES);
            return;
        }
    }
}
