C# Hello World
이전 C나 C++에서 맞이했던 Hello World를 C# 에서도 맞이하는군요
우선 Hello World를 작성하기 전 개발 환경을 만든 소스 코드를 전부 지우시고
아래의 소스 코드를 직접 타이핑 하세요.


using System;
using static System.Console;

namespace CSharp
{
    class HelloWorld
    {
        static void Main()
        {
            Console.WriteLine("Hello World");
            return;
        }
    }
}

구조가 C나 C++에 비해 뭔가 많네요. 단지 Hello World를 출력하는것 뿐인데 말이죠.

using은 하나의 키워드(Keyword)입니다.
C++에서는 같은 전처리문을 썻지만 C# 에서는 키워드를 사용합니다.
그리고 문장의 끝은 세미콜론(;)으로 나타내죠

그렇다면 using System;는 키워드 using으로 인해 System을 사용한다는 뜻입니다.

그리고 namespace도 궁금하실텐데 이 부분은 다음 기회에 설명 하도록 하겠습니다.

그리고 이 namespace를 구성하는 구조는 어떤것들이 있는지 알아 보도록 하죠.

namespace의 구조는 클래스, 구조체, 인터페이스 등으로 구성이 되며
namespace CSharp 은
CSharp이라는 이름을 가진 하나의 집합이라고 볼 수 있습니다.



그리고 C# 에서의 클래스는 C++에서의 클래스와는 좀 다른 개념인데요.
C++에서는 클래스를 인스턴스(할당)해줘서 객체를 만든다는 개념이라면
C# 은 프로그램을 구성하는 기본 단위이며 메소드(Method)라고 합니다.
(메소드는 데이터와 데이터를 처리하는 기능입니다)
개념만 다를 뿐 C++과 닮은점이 몇가지 있습니다. 이는 나중에
Class에 대해서 깊게 알아볼때 설명하겠습니다.

그럼 제일 중요한 부분을 보도록 하죠.
우리는 C나 C++을 할때 항상 main이라는 함수가 존재 해야했습니다.
main은 프로그램의 시작을 알리는 부분이며 모든 언어가 main을 약속처럼 사용하죠.
그것은 C# 도 다르지 않습니다. 다만 형태가 많이 다를 뿐이죠.

static void Main(string[] args)

이는 메소드이며 시작점입니다. 이 메소드가 끝나면 프로그램도 끝나죠.
그리고 메소드는 데이터와 데이터를 처리하는 기능이기도 하며
C에서는 이를 함수라고 불렀습니다. 여기서 다시 메소드에 대해서 자세히 짚고 넘어가겠습니다.

메소드(Method)
데이터와 데이터를 처리하는 기능이다.
절차 지향 프로그래밍에서는 함수라고 불렀으며
객체 지향 프로그래밍에서의 메소드, 객체의 일부다

앞으로 많이 나올 단어니 꼭 알아 두세요.
그리고 다른 언어의 main과는 달리 static이라는 키워드가 붙었습니다.
저도 직접 사용해 본적은 없고 기능만 살짝 알던 키워드인데 C# 에 오자마자 마주치네요.

이는 정적 변수를 선언할때 쓰이는 키워드인데요.
다른 변수와 달리 프로그램이 실행되는 동시에 메모리에 할당을 시킵니다.

main에 static키워드가 붙는 이유는 C# 의 컴파일 방식 때문인데요.
프로그램이 시작될 때 CRL이 프로그램의 진입점을 찾지 못합니다.
그래서 static으로 실행됨과 동시에 할당해주어 진입점을 찾게 해줍니다.

소스 코드를 전체적으로 해석하자면
1. using키워드로 System, System Console을 사용한다.
2. namespace CSharp의 집합에 Class HelloWorld라는 클래스가 있으며
3. 메소드 Main이 있다, 메소드 Main은 시작점이자 끝이고
4. Console.WriteLine로 출력한다.
