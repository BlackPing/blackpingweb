C# CLR
C# 으로 만든 프로그램은 CLR(Common Language Runtime)위에서 실행이 됩니다.

CLR은 C# 을 프로그램으로 만들기 전에 중간 과정을 거친다고 보시면 됩니다. 특히 C# 의 언어는 C와 C++과는 틀린점이 있는데요.
C와 C++은 컴파일을 진행해서 결국엔 프로그램을 바로 만들어냅니다.
하지만 C# 은 CLR이라는 중간 과정을 거처서 다른 언어 사용할 수 있게 지원을 하죠 이때 서로 다른 언어들이 만나는 지점을 IL이라는 중간 언어이며
이 언어로 쓰여진 코드를 CLR이 다시 자신이 실행한 플랫폼에 맞춰서 컴파일을 해줍니다.

한 마디로 CLR은 따로 컴파일을 해서 실행을 시키고 그걸 해당 플랫폼에 다시 맞춰서 컴파일을 해주게하는 기능을 가지고 있습니다.
C# 컴파일 -> IL생성 -> CLR -> 실행파일
이때 IL생성 후 CLR이 읽는 작업을 JIT(Just In Time)컴파일이라고 합니다.
그렇다면 결국 C# 컴파일 -> JIT 컴파일 -> 실행파일
컴파일을 2번 진행하네요.

JIT 컴파일이란?
실행에 필요한 코드를 실행할 때마다 실시간으로 컴파일해서 실행

결국 위에서 설명한걸 종합적으로 정리를 하자면
최적화를 위해 컴파일을 2번하는 겁니다.
실제 내용은 컴파일 비용을 좀더 부담하고 기계에 최적화되어 최고의 성능을 내는 방식의 컴파일이라고 하지만 이렇게까지 외우면
외워봤자 금방 까먹을거 같네요.

그래서 그냥 최적화를 위해 컴파일을 2번한다고 생각하세요.

그리고 CLR에는 중간 언어를 생성하고 실행하는 것외에 또 다른 기능이 있는데요.

프로그램의 오류가 발생했을 때 이를 처리하도록 도와주는 기능, 언어간의 상속 지원, COM과의 상호 운영성 지원, 자동 메모리 관리 등의 기능을 제공 합니다.
자동 메모리 관리 기능은 가비지 컬렉션(Garbage Colletion)이라고 합니다. (가비지 컬렉션은 C# 하던 친구한테 한번쯤 들어본 단어네요)
프로그램에서 더 이상 사용하지 않는 메모리를 쓰레기로 간주하고 수거하는 기능입니다.

좀더 쉽게 말하자면 C나 C++은 동적으로 할당을 해주고 이 메모리가 쓰이지 않는다면 할당을 해제해줘서 쓸모없는 메모리를 안쓰도록 직접 수거했죠.
하지만 가비지 컬렉션은 그러한 작업 없이 안쓰면 자동으로 할당을 해제해준다고 생각하시면 됩니다.
