프로그램 구조
프로그램은 어떤 프로그래밍 언어로 작성 되었든 내부적인 구조는 함수들로 구성되어 있으며
CPU에서 명령을 수행하려면 수행하려는 주소 영역이 메모리에 적재 되어야 한다.

프로그램의 주소 영역은 코드, 데이터, 스택, 힙이다.


코드 영역 : 기계어 명령으로 저장되는 부분
제어문이나 상수문이 저장되며 프로그램이 끝날 때까지 할당
데이터 영역 : 전역 변수 등 프로그램이 사용하는 데이터 저장
(전역 변수, static 변수) 프로그램이 끝날 때까지 할당
스택 : 함수가 호출될 때 호출된 함수의 수행을 마치고 복귀할 주소 및 데이터를 임시로 저장
지역 변수와 매개 변수가 저장되는 메모리 영역
힙 : 프로그래머가 동적으로 할당하는 메모리 영역

프로세스의 주소 공간은 코드 데이터 스택 힙으로 구성되어 있으며
각 프로그램마다 독자적으로 존재하는 주소 공간을 가상 메모리 또는 논리적 메모리 라고 부른다.

프로그램 실행
실행 파일은 메모리에 적재가 되는데 메모리가 적재될 때 한번에 적재되지 않고
일부분만 메모리에 적재가 된다. 나머지는 디스크의 특정 영역에 내려가 있는데
이는 한정적인 메모리 공간을 효율적으로 사용하기 위해서다.

이 내용을 정리하자면
프로그램 주소 공간 중 당장 CPU의 수행이 필요한 부분만 메모리에 적재 되고
그렇지 않으면 메모리의 연장 공간인 스왑 영역에 내려 놓는다.

이와 같다.

프로그램의 실행 상태를 컴퓨터에서 중요한 의미가 있는데
실행 파일이 메모리에 적재가 돼있는 상태
프로그램이 CPU에 할당받고 기계 명령을 수행하는 상태

바로 위의 두가지 경우다.

프로그램은 실행이 될때 짧은 시간 단위로 CPU를 다른 프로그램과 나누어 쓰는데
이는 프로그램들이 메모리에 동시에 적재되어 있어서 그렇다.
이러한 동작을 시분할 시스템이라고 한다.
