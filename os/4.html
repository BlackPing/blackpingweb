인터럽트(Interrupt)
하드웨어 인터럽트와 트랩을 포함한 개념
트랩이란 소프트웨어가 발생시키는 인터럽트

인터럽트는 하드웨어 컨트롤러가 CPU에 붙어있는 인터럽트 라인을 통해서
신호를 받아서 현재 수행하던 프로세스를 중단하고 인터럽트 처리를 수행
하드웨어 인터럽트
하드웨어 컨트롤러가 CPU의 서비스를 요청하기 위해 발생시키는 인터럽트

인터럽트의 필요 이유
CPU는 특별한 일이 없으면 현재 수행중인 프로세스 다음 명령을 순차적으로 수행하다가
인터럽트 신호가 들어오면 인터럽트 처리를 하는데 이는 한 프로세스의 CPU 독점을 막기 위해 인터럽트 메커니즘을 사용하는 것이다.

특정 프로세스가 CPU 독점을 막기 위해 운영 체제는 타이머 인터럽트를 사용한다.
원리는 세팅된 시간이 되면 CPU에게 인터럽트를 발생시킨다.

인터럽트는 한 프로세스의 CPU 독점을 막기 위해 사용

소프트웨어 인터럽트
예외 상황(exception)
프로그램이 허용되지 않은 연산을 수행하려고 할 때 자동적으로 발생
하드웨어 RESET

0으로 나누는 연산
정해진 메모리 밖의 엑세스
소프트웨어 인터럽트
컴퓨터 시스템의 기능 장치로 부터 발생하는 인터럽트

UART

기타 장치

시스템 콜(system call)
프로세스가 운영 체제의 서비스를 요청하기 위해 커널의 함수를 호출하는 것
사용자 프로세스가 특권명령을 사용하기 위해 시스템 콜을 사용

인터럽트 서비스 루틴(interrupt service routine)
인터럽트가 발생했을 때 주행 시키는 루틴

인터럽트 벡터(interrupt vector)
인터럽트를 거는 장치에 할당된 두 개의 주 기억 장치 두 개의 주소는 서비스 루틴에 대한 처리 장치 상태와 서비스 루틴의 시작 주소를 포함하고 있다.
인터럽트 발생시 처리해야 할 루틴의 주소를 보관 하는 테이블

인터럽트 핸들러(interrupt handler)
운영 체제 코드 부분에는 각종 인터럽트 별로 처리해야 할 내용이 이미 프로그램되어 있어 이 부분을 인터럽트 서비스 루틴 또는 인터럽트 핸들러 라고 부른다.
실제 인터럽트를 처리하기 위한 루틴.

루틴(routine)
특정한 작업을 실행하기 위한 일련의 명령. 프로그램의 일부 혹은 전부를 이르는 경우에 쓴다
